# CLAUDE.md — skill-plugin-audit

## Project Overview

Static GitHub Pages site that archives and serves **daily Claude plugin/skill/MCP audit reports** as HTML files. No backend, no build step — deploy script copies HTML → `docs/` and pushes to `main`.

- **Live index**: `https://thitic9203.github.io/skill-plugin-audit/`
- **Latest report** (permalink): `https://thitic9203.github.io/skill-plugin-audit/latest.html`
- **Per-day archive**: `https://thitic9203.github.io/skill-plugin-audit/reports/YYYY-MM-DD.html`

## Tech Stack & Architecture

- **Hosting**: GitHub Pages (serves `docs/` folder of `main` branch)
- **No framework, no build tool** — pure HTML/CSS, deployed via `git push`
- **Deploy script**: `scripts/deploy-report.sh` — shell script, macOS `date -j` format

## Directory Structure

```
docs/
  index.html          # Landing page — auto-regenerated by deploy-report.sh
  latest.html         # Always the newest report (overwritten on each deploy)
  reports/
    YYYY-MM-DD.html   # Immutable daily archive
scripts/
  deploy-report.sh    # One-command deploy
README.md
CLAUDE.md             # This file
```

## Key Concepts

- **Report source**: HTML files generated externally by a scheduled task (not produced in this repo)
- **Deploy flow**: external tool generates `daily-skill-plugin-audit-YYYY-MM-DD.html` → `deploy-report.sh` ingests it
- **`latest.html`**: always overwritten with the newest report; used as a stable Slack-pinnable URL
- **`index.html`**: auto-regenerated each deploy — lists all reports newest-first; do not hand-edit

## Common Tasks

```bash
# Deploy a new report
./scripts/deploy-report.sh /path/to/daily-skill-plugin-audit-2026-04-15.html

# Check GitHub Pages is serving correctly
curl -sI https://thitic9203.github.io/skill-plugin-audit/ | grep HTTP

# List all archived reports
ls docs/reports/
```

## Environment & Config

- **GitHub Pages**: Settings → Pages → Branch: `main`, Folder: `/docs`
- **Auth**: `gh auth login` must be active for `git push` to succeed
- **macOS only**: `deploy-report.sh` uses `date -j -f` (BSD date) — not compatible with GNU `date`

## Known Constraints & Gotchas

- `index.html` is fully regenerated on every deploy — manual edits will be overwritten
- `latest.html` is always overwritten — it is not an archive; the archive is `reports/YYYY-MM-DD.html`
- GitHub Pages takes 30–60 s to refresh after each push
- Deploy script exits non-zero if the source HTML path is missing — always verify path before running
- Reports are standalone HTML with all styles inlined — no external CSS dependencies at runtime
